## 二、产品定位与目标

### 2.1 产品定位

一款面向 22408 考研考生的 **Web 端备考计划管理工具**，帮助考生可视化管理从 8 月到初试的全阶段学习进度，追踪每日任务完成情况，量化备考效果。

### 2.2 核心目标

1. **计划可视化**：以日历 / 甘特图 / 进度条等形式清晰展示各阶段任务
2. **任务管理**：每日任务打卡，支持完成 / 未完成状态切换
3. **进度追踪**：实时统计各科目复习进度百分比
4. **数据持久化**：本地存储学习记录，刷新不丢失
5. **自适应计划**：可根据实际进度动态调整后续任务

### 4.2 核心功能详细说明

#### F1：考研倒计时

- 展示距离初试天数，精确到天
- 支持自定义考试日期
- 在各页面顶部常驻显示

#### F2：阶段进度管理

- 可视化展示四个备考阶段
- 当前阶段高亮，已完成阶段标记 ✅
- 点击阶段可展开查看详细周计划

#### F3：每日任务打卡

- 根据当前所处阶段自动生成当日推荐任务
- 支持手动添加/编辑/删除任务
- 任务可标记为：⬜ 未开始 / 🟡 进行中 / ✅ 已完成
- 打卡后自动更新科目进度

#### F4：番茄钟计时器

- 默认 25 分钟工作 + 5 分钟休息
- 支持自定义时长
- 计时结束提醒（声音 + 通知）
- 累计学习时长自动记录

#### F5：科目进度追踪

- 四科独立进度条
- 进度 = 已完成任务数 / 总任务数 × 100%
- 颜色编码：红色（< 30%）→ 黄色（30-70%）→ 绿色（> 70%）

#### F6：真题得分管理

- 录入每套真题各科得分
- 自动计算总分及各科平均分
- 折线图展示得分趋势
- 目标分数线参考（南邮历年分数线）

#### F7：错题本

- 支持录入错题（题目描述 + 所属科目 + 知识点标签）
- 支持上传错题图片
- 标记重做状态
- 统计各知识点错误频率

#### F8：数据持久化

- 使用 LocalStorage / IndexedDB 本地存储
- 支持 JSON 格式数据导出/导入
- 可选：接入后端数据库实现云同步

#### F9：每日时间段任务表（⭐ 核心功能）

> 以时间轴形式展示每天各时段应完成的具体任务，每项任务附带明确的**达标要求**，考生可逐项打钩确认完成情况。

##### 9.1 功能概述

- 按当前所处**备考阶段**自动加载对应的每日时间表模板
- 以**时间轴 / 课程表**形式纵向排列，每个时段一行
- 每个时段包含：时间区间、科目、具体任务内容、达标要求、完成打钩
- 顶部显示**当日完成率**（已完成 / 总任务数）
- 支持切换日期查看历史完成记录

##### 9.2 时段任务卡片结构

每个时段任务以卡片形式展示，包含以下字段：


##### 9.3 完成状态机制

| 状态 | 图标 | 触发条件 | 说明 |
|------|------|----------|------|
| 未开始 | ⬜ | 默认状态 | 当前时段尚未开始 |
| 进行中 | 🔵 | 手动切换 | 正在学习该时段内容 |
| 全部达标 | ✅ | 所有达标要求均打钩 | 完美完成，计入100%进度 |
| 部分完成 | 🟡 | 达标要求打钩数 ≥ 50% | 基本完成，计入对应比例进度 |
| 未完成 | ❌ | 达标要求打钩数 < 50% 且时段已过 | 需要补学，标红提醒 |
| 跳过 | ⏭️ | 手动标记 | 主动跳过（如身体不适），不计入统计 |

##### 9.5 打钩确认交互逻辑

```
用户操作流程：

1. 进入「每日时间表」页面
   └─→ 自动定位到当前时段（高亮显示）

2. 展开某个时段任务卡片
   └─→ 查看具体任务内容和达标要求列表

3. 逐项打钩达标要求
   ├─→ 每打一个钩，进度条实时更新
   ├─→ 达标要求全部打钩 → 自动标记为 ✅ 全部达标
   ├─→ 打钩 ≥ 50% → 标记为 🟡 部分完成
   └─→ 时段过期且 < 50% → 标记为 ❌ 未完成（标红）

4. 当日所有时段处理完毕
   ├─→ 顶部显示「今日完成率 XX%」
   ├─→ 完成率 ≥ 90% → 🎉 弹出鼓励动画
   ├─→ 完成率 70%-89% → 💪 提示继续加油
   └─→ 完成率 < 70% → ⚠️ 提示需要调整状态

5. 支持手动添加备注
   └─→ 每个时段可写自由文字备注（如：今天计组没看完，明天补上）
```

### 5.3 核心数据结构

```typescript
// 科目枚举
enum Subject {
  DataStructure = "数据结构",
  ComputerOrganization = "计算机组成原理",
  OperatingSystem = "操作系统",
  ComputerNetwork = "计算机网络",
}

// 阶段枚举
enum Phase {
  Foundation = "强化基础",      // 8月-9月中
  Enhancement = "强化提升",     // 9月中-10月底
  PastExams = "真题冲刺",       // 11月-12月初
  FinalSprint = "考前冲刺",     // 12月初-考前
}

// 任务
interface Task {
  id: string;
  title: string;
  subject: Subject;
  phase: Phase;
  date: string;             // YYYY-MM-DD
  status: "todo" | "doing" | "done";
  estimatedMinutes: number;
  actualMinutes?: number;
  notes?: string;
}

// 真题记录
interface ExamRecord {
  id: string;
  year: number;             // 真题年份
  date: string;             // 做题日期
  scores: {
    dataStructure: number;
    computerOrganization: number;
    operatingSystem: number;
    computerNetwork: number;
  };
  totalScore: number;
  notes?: string;
}

// 错题
interface Mistake {
  id: string;
  subject: Subject;
  tags: string[];           // 知识点标签
  description: string;
  imageUrl?: string;
  source: string;           // 来源（如：王道P123、2023真题第3题）
  createdAt: string;
  reviewCount: number;
  lastReviewAt?: string;
  mastered: boolean;
}

// 每日学习记录
interface DailyLog {
  date: string;
  totalMinutes: number;
  pomodoroCount: number;
  tasksCompleted: number;
  tasksTotal: number;
}

// ========== 每日时间段任务表相关类型 ==========

// 达标要求项
interface Criterion {
  id: string;
  text: string;              // 达标要求描述
  checked: boolean;           // 是否已打钩确认
}

// 时段任务完成状态
type SlotStatus = 
  | "not_started"   // ⬜ 未开始
  | "in_progress"   // 🔵 进行中
  | "completed"     // ✅ 全部达标
  | "partial"       // 🟡 部分完成
  | "failed"        // ❌ 未完成
  | "skipped";      // ⏭️ 跳过

// 单个时段任务
interface TimeSlotTask {
  id: string;
  startTime: string;          // "08:00"
  endTime: string;            // "09:30"
  subject: Subject | "休息" | "英语" | "数学" | "政治" | "复盘" | "其他";
  icon: string;               // 科目对应 emoji
  title: string;              // 任务简称
  description: string;        // 具体任务内容
  criteria: Criterion[];      // 达标要求列表
  status: SlotStatus;         // 完成状态
  notes?: string;             // 用户自由备注
  isRestPeriod: boolean;      // 是否为休息时段（不计入统计）
}

// 某一天的完整时间表
interface DailySchedule {
  date: string;               // YYYY-MM-DD
  phase: Phase;               // 所属阶段
  slots: TimeSlotTask[];      // 时段任务列表
  completionRate: number;     // 当日完成率（0-100）
  totalCriteria: number;      // 总达标项数
  checkedCriteria: number;    // 已完成达标项数
}

// 时间表模板（按阶段预设）
interface ScheduleTemplate {
  phase: Phase;
  slots: Omit<TimeSlotTask, "id" | "status" | "notes">[];
}
```

## 七、考试科目重点内容速查

供程序内置「知识点清单」功能使用：

### 数据结构（45 分）

- 线性表：顺序表与链表操作、栈和队列应用
- 树：二叉树遍历（前/中/后/层序）、线索二叉树、哈夫曼树
- 图：邻接表/矩阵、BFS/DFS、Prim/Kruskal、Dijkstra/Floyd
- 查找：顺序/折半/分块查找、BST、AVL、B树/B+树、散列表
- 排序：直接插入、希尔、冒泡、快排、简单选择、堆排序、归并、基数排序

### 计算机组成原理（45 分）

- 数据表示：原码/反码/补码/移码、IEEE 754 浮点数
- 存储系统：Cache 映射（直接/全相联/组相联）、虚拟存储器
- 指令系统：指令格式、寻址方式
- CPU：数据通路、微程序/硬布线控制器、指令流水线
- 总线与 I/O：总线仲裁、程序查询/中断/DMA

### 操作系统（35 分）

- 进程管理：进程状态转换、进程同步（PV操作）、死锁（银行家算法）
- 内存管理：页式/段式/段页式、页面置换算法、内存分配算法
- 文件系统：目录结构、文件分配方式、磁盘调度算法
- I/O 管理：I/O 控制方式、假脱机（SPOOLing）

### 计算机网络（25 分）

- 体系结构：OSI 七层 / TCP/IP 四层模型
- 数据链路层：差错控制、流量控制、CSMA/CD
- 网络层：IP 编址、子网划分、路由协议（RIP/OSPF）
- 传输层：TCP（三次握手/四次挥手/滑动窗口/拥塞控制）、UDP
- 应用层：DNS、HTTP、SMTP/POP3、FTP

## 附录：Cursor 开发提示

在 Cursor 中开发此项目时，建议使用以下 prompt 策略：

1. **初始化项目**：让 Cursor 用 `npm create vite@latest` 创建 React + TypeScript 项目
2. **分模块开发**：按照上述目录结构逐模块生成代码
3. **数据优先**：先让 Cursor 生成 `planData.ts` 中的完整阶段数据
4. **组件驱动**：先开发通用组件（进度环、任务卡片），再组装页面
5. **Prompt 示例**：

```
请根据以下 PRD 中的数据结构和计划数据，
创建 Dashboard 页面，包含：
1. 距离 2026年12月考试的倒计时
2. 四个科目的进度环（使用 Recharts RadialBarChart）
3. 今日任务列表（可打卡切换状态）
4. 使用 TailwindCSS 美化样式
5. 数据存储使用 Zustand + localStorage
```
